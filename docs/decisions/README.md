# 의사결정 로그 (Decision Log)

자율 개발 루프(`/dev:auto-dev`)가 **스스로 결정하면 안 되는 지점**에 도달했을 때 사람에게 넘기는 곳입니다. 에이전트는 강행하지 않고 여기에 질문/결함 리포트를 남긴 뒤 **카카오 알림을 보내고 루프를 정지**합니다.

## 언제 여기에 기록되나 (정지 트리거)

- **비가역 결정**: DB 스키마 변경, 외부 서비스 연동, 데이터 삭제 등 되돌리기 어려운 선택.
- **모호한 요구**: 스펙만으로 판단이 갈리는 지점(코드·문서 확인으로도 해소 안 됨).
- **QA 반복 실패**: `qa-tester`가 최대 재시도(기본 2회) 후에도 fail — 결함 리포트를 남긴다.

## 흐름

1. 에이전트가 `PENDING-<슬러그>.md`를 이 디렉토리에 생성한다.
2. 카카오로 알림이 간다(`scripts/kakao/send.mjs`).
3. 루프가 정지한다(진행 강행 금지).
4. **사람**이 해당 파일의 `## 결정` 섹션을 채우고, 상태를 `RESOLVED`로 바꾼다(파일명을 `RESOLVED-<슬러그>.md`로 변경하거나 상단 상태 표기).
5. 다음 `/dev:auto-dev` 실행이 **PENDING 파일을 먼저 확인**한다 — 있으면 해당 결정을 반영해 재개하고, 없으면 다음 태스크로 진행한다.

## 파일 템플릿

새 안건은 아래 형식으로 `PENDING-<슬러그>.md`에 작성합니다.

```markdown
# <한 줄 제목>

- **상태**: PENDING
- **작성**: <날짜> · auto-dev
- **관련 태스크**: <Task/Phase 또는 shrimp task id>

## 맥락
<무엇을 하다가 왜 멈췄는지 배경>

## 막힌 지점 / 질문
<사람이 답해야 할 구체적 질문. 또는 QA 결함이면 재현 절차(경로·조작·기대 vs 실제·콘솔로그)>

## 선택지
- **A**: <설명 / 트레이드오프>
- **B**: <설명 / 트레이드오프>

## 기본 추천
<에이전트가 권장하는 선택과 근거. 사람이 다른 선택을 하면 그것을 따른다.>

## 결정
<사람이 여기에 답을 적는다. 채워지면 상태를 RESOLVED로 바꾼다.>
```

## 규칙

- 시크릿(토큰·키)은 이 문서에도 남기지 않는다(커밋 가드가 차단).
- 문서는 한국어로 작성한다.
- `RESOLVED-*.md`는 기록으로 보존한다(삭제하지 않음).
