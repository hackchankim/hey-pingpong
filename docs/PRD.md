# 생활체육 리그/랭킹 관리 플랫폼 MVP PRD

## 🎯 핵심 정보

**목적**: 사설 체육관(탁구/테니스 동호회)에서 수기로 이루어지던 대진표 작성과 점수 계산을 디지털화해 대회 운영을 자동화한다.
**사용자**: 탁구/테니스 동호회에 소속된 생활체육 동호인과 구장 운영자(관장/총무).

## 🚶 사용자 여정

```
1. 로그인 / 회원가입
   ↓

2. 내 클럽 페이지
   ↓ [소속 클럽 확인]

   [클럽 없음] → 클럽 생성 페이지 또는 클럽 가입 페이지 → 클럽 홈 페이지
   [클럽 있음] → 클럽 홈 페이지
   ↓

3. 클럽 홈 페이지
   ↓ [메뉴 선택]

   [멤버 관리]  → 멤버 관리 페이지 (관리자)
   [대회]       → 대회 목록 페이지 → 대회 상세 페이지 → 대진표 페이지
   [랭킹]       → 랭킹 페이지
   ↓

4. 대진표 페이지
   ↓ [경기 점수 입력]

   점수 입력 완료 → 승자 자동 판정 → 다음 라운드 자동 배정(토너먼트) → 랭킹 자동 갱신
   ↓

5. 랭킹 페이지 → 클럽 순위표 및 내 레이팅 추이 확인
```

## ⚡ 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F001** | 클럽 생성 | 구장 운영자가 새 구장 공간을 만들고 초대코드를 발급 | 구장별 운영의 시작점 | 클럽 생성 페이지 |
| **F002** | 초대코드로 클럽 가입 | 회원이 초대코드를 입력해 구장 커뮤니티에 합류 | 구장별 폐쇄형 운영의 핵심 전제 | 클럽 가입 페이지 |
| **F003** | 클럽 멤버 관리 | 멤버 목록 확인, 관리자 권한 부여/해제 | 대회 운영 권한 통제 | 멤버 관리 페이지 |
| **F004** | 대회 생성 | 대회명, 대진 방식(풀리그/토너먼트), 참가 정원 설정 | 대회 운영의 출발점 | 대회 목록 페이지 |
| **F005** | 대회 참가자 관리 | 참가 신청/승인 처리, 참가자 등록 마감, 이미 승인된 참가자 실격 처리, 본인 기권 | 대진표 생성 전 필수 단계 및 대회 도중 예외 운영 | 대회 상세 페이지 |
| **F006** | 자동 대진표 생성 | 참가자 확정 시 선택한 형식(풀리그/싱글 토너먼트/더블 엘리미네이션)에 맞춰 대진표를 자동 생성 | 수기 대진표 작성을 대체하는 핵심 가치 | 대진표 페이지 |
| **F007** | 경기 점수 입력 | 경기별 세트 점수를 입력하면 승패를 자동 판정 | 수기 점수 계산을 대체하는 핵심 가치 | 대진표 페이지 |
| **F008** | 대진 자동 진행 | 토너먼트에서 승자가 다음 라운드로(더블 엘리미네이션은 패자도 패자조로) 자동 배정, 전 경기 종료 시 대회 완료 처리 | 대회 운영 자동화의 핵심 | 대진표 페이지 |
| **F009** | 랭킹 자동 산정 | 경기 결과마다 레이팅(ELO 방식)이 자동으로 갱신됨 | 공식 성적 기반 승부욕 자극 | 대진표 페이지, 랭킹 페이지 |
| **F010** | 랭킹 조회 | 클럽 내 순위표와 내 레이팅 변동 추이를 확인 | 참여 동기 제공 | 랭킹 페이지 |
| **F011** | 대회 취소 | 관리자가 진행 전/중인 대회를 취소 처리(완료된 대회는 취소 불가) | 오운영·인원 미달 등 예외 상황 대응 | 대회 상세 페이지 |

|------|------|----------|
| id | 고유 식별자 | UUID |
| club_id | 소속 클럽 | → clubs.id |
| name | 대회명 | text |
| format | 대진 방식(풀리그/토너먼트) | text |
| status | 진행 상태 | text |

### tournament_participants (대회 참가자)
| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 고유 식별자 | UUID |
| tournament_id | 소속 대회 | → tournaments.id |
| user_id | 참가자 | → profiles.id |
| status | 신청 상태(신청/승인/취소) | text |

## 🛠️ 기술 스택 (최신 버전)

### 🎨 프론트엔드 프레임워크
- **Next.js 16** (App Router, Cache Components) - React 풀스택 프레임워크
- **TypeScript 5** - 타입 안전성 보장
- **React 19** - UI 라이브러리 (최신 동시성 기능)

### 🎨 스타일링 & UI
- **Tailwind CSS v3.4** - 유틸리티 CSS 프레임워크
- **shadcn/ui** - 고품질 React 컴포넌트 라이브러리
- **Lucide React** - 아이콘 라이브러리

### 📝 폼 & 검증
- **React Hook Form 7.x** - 폼 상태 관리
- **Zod** - 스키마 검증 라이브러리

### 🗄️ 백엔드 & 데이터베이스
- **Spring Boot 3.3(LTS) + Kotlin** - 클럽/대회/대진표/랭킹 비즈니스 로직을 전담하는 API 서버, 모노레포 내 `api/` 디렉토리로 Next.js에서 단계적으로 분리 중
- **Spring Data JPA(Hibernate)** - ORM 및 트랜잭션 처리, 잠금·벌크 처리가 필요한 구간은 네이티브 쿼리로 보완
- **PostgreSQL** - 관계형 데이터베이스
- **Supabase Auth** - 회원가입/로그인/OAuth 등 인증 전용으로 유지, Spring Security OAuth2 Resource Server가 발급된 JWT를 검증

### 🚀 배포 & 호스팅
- **GCP / AWS** - 프론트엔드(Next.js)와 API 서버(Spring Boot)를 각각 컨테이너로 배포

### 📦 패키지 관리
- **npm** - 의존성 관리
